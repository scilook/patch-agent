# PRD: Ubuntu Vulnerability Patch Management Solution (MVP)

📋 **헤더 정보**
- **Product Title:** Ubuntu 22.04 Vulnerability Discovery & Patch Agent (PoC)
- **Document Version:** 1.0.0
- **Author:** Son Hyeongrok (Tech Lead)
- **Last Updated:** 2026-04-22

🎯 **프로젝트 정의 섹션**

**1. Overview**
본 프로젝트는 Ubuntu 22.04 시스템의 보안 취약점(CVE/KVE)을 자동으로 탐지하고, 지능적인 패키지 매칭을 통해 안전한 패치 업데이트를 수행하는 최소 기능 제품(MVP) 구축을 목표로 함.

**2. Background**
최신 연구에 따르면 자동화된 패치 관리는 배포 시간을 80% 단축하고 시스템 가동 중지 시간을 87.5% 줄일 수 있음. 기존 수동 방식의 한계를 극복하고, 특히 CPE 양식과 dpkg 양식 간의 불일치를 해결하는 인텔리전트 매칭 시스템이 필요함.

**3. Objectives and Goals**
- **데이터 자동화:** NVD API 2.0 및 KVE 데이터를 SQLite3에 효율적으로 적재.
- **지능적 매칭:** Ubuntu OVAL 데이터를 활용하여 CPE와 실제 설치 패키지 간의 매칭 정확도 향상.
- **보안 강화:** `patch-agent` 전용 계정과 최소 권한 원칙(PoLP)을 적용하여 시스템 안정성 확보.
- **효율적 완화:** 수동 패치 대비 20% 이상의 관리 효율성 개선.

🔧 **기능 명세 섹션**

**4. Key Features**

**4.1 Vulnerability Data Management (DB)**
- SQLite3를 활용한 가벼운 데이터베이스 구축.
- **Vulnerabilities 테이블:** CVE ID, 설명, CVSS 점수, 발행일 저장.
- **Affected_Packages 테이블:** 매칭을 위한 제품명, 취약 버전 범위(Version Start/End) 저장.
- API UPSERT 로직: `lastModStartDate` 파라미터를 통한 증분 업데이트.

**4.2 Intelligent Matching Engine**
- **OVAL Translator:** Ubuntu OVAL 데이터를 파싱하여 CPE 양식을 dpkg 패키지 명칭으로 변환.
- **Version Comparator:** 단순 문자열 비교가 아닌 `apt_pkg` 모듈 또는 `dpkg --compare-versions`를 통한 리비전 단위 버전 판별.
- **Contextual Discovery:** `dpkg-query` 결과를 기반으로 현재 시스템에 설치된 취약 패키지 자동 식별.

**4.3 Secure Patch Execution**
- **patch-agent 계정:** 쉘 접근이 제한된 서비스 계정 기반 실행.
- **Selective Sudo:** `/etc/sudoers.d/patch-agent` 설정을 통해 `apt-get update` 및 `install --only-upgrade` 명령만 허용.
- **Audit Logging:** 모든 패치 수행 이력을 JSON 포맷의 표준 로그 파일로 생성.

👥 **권한 및 제약사항**

**5. User Roles and Permissions**
- **Root/Admin:** 시스템 초기 설정 및 `patch-agent` 권한 부여.
- **patch-agent:** 취약점 스캔, DB 비교, 허가된 패키지 업데이트 수행 (비로그인 계정).

**6. API Contracts & Data Flow**
- **Input:** NVD API 2.0 (JSON), Ubuntu OVAL (XML).
- **Process:** Fetch ➡️ Parse ➡️ SQLite3 Store ➡️ Local Scan ➡️ Match ➡️ Patch.
- **Output:** 패치 결과 및 취약점 완화 성공률 리포트 (JSON).

⚙️ **기술적 요구사항**

**7. Non-Functional Requirements**
- **보안성:** 전체 프로세스 중 `root` 권한 사용 최소화.
- **정확성:** 가짜 양성(False Positive) 발생률 3% 미만 유지.
- **추적성:** 모든 의사결정 근거(XAI 원칙)를 로그에 기록.

**8. Assumptions and Dependencies**
- OS 환경: Ubuntu 22.04 LTS 전용.
- 외부 의존성: NVD API 및 Ubuntu OVAL 피드 접근 가능 여부.
- 라이브러리: Python 3.10+, SQLite3, `python3-apt`.

📊 **성공 지표 및 범위**

**9. Success Metrics**
- **완화 성공률:** 탐지된 취약점의 90% 이상 성공적 패치.
- **안전성:** 패치 과정 중 시스템 오염이나 권한 남용 사례 0건.
- **정확도:** OVAL 매핑을 통한 패키지 이름 일치율 95% 이상.

**10. Out of Scope**
- 커널 라이브 패칭 (Kpatch/KGraft 등).
- GUI 기반 대시보드 (본 MVP는 CLI 및 로그 기반).
- 사용자 직접 로그인 및 인증 플로우 (OAuth 등).
- 타 OS(CentOS, Arch 등) 지원.
